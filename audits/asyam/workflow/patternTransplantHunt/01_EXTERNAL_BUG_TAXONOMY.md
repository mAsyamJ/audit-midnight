# External Bug Taxonomy

Each category is a historical prompt mapped to a local test strategy. It is not a vulnerability claim.

| Category | General bug shape | Typical impact | Common false positives | Midnight surfaces | Local test strategy |
| --- | --- | --- | --- | --- | --- |
| Access control / authorization | A capability reaches a caller or call context broader than the intended authority. | unauthorized state change or custody loss | admin-only behavior | setIsAuthorized, role setters, ratifiers | compose direct and signed authorization with relayer context |
| Signature / permit / replay / intent mismatch | A valid signature authorizes a materially different economic effect. | token or position loss | explicitly signed capability | EcrecoverAuthorizer, EcrecoverRatifier, HashLib, Permit2, ERC2612 | bind signer, operation, receiver, market, nonce, and sequence |
| Accounting / shares / units / debt / credit | Equivalent value is represented by fields that drift under ordering. | insolvency or unfair withdrawal | rounding dust | marketState, position, totalUnits, withdrawable | compare aggregate accounting before and after multi-step flows |
| Rounding / precision / dust escalation | Small error crosses a later material decision threshold. | non-dust branch flip | dust-only delta | TakeAmountsLib, ConsumableUnitsLib, TickLib, fee math | place values one wei around thresholds and continue the lifecycle |
| Oracle / price / stale data / manipulation | Price trust or liveness assumptions leak across a consent boundary. | bad liquidation or frozen funds | user chose malicious market | IOracle, liquidate, health checks | prove involuntary exposure to the bad oracle |
| Liquidation / bad debt / solvency | Liquidation ordering realizes loss before recoverability is exhausted. | avoidable lender loss | documented liquidation incentive | liquidate, lossFactor, badDebt | compare equivalent collateral-order permutations |
| Callback / reentrancy / cross-function state | External control observes or consumes intermediate state. | custody or health break | malicious callback harms itself | take, liquidate, flashLoan, ICallbacks | cross a new state boundary, not direct reentry |
| Router / aggregator / periphery / multicall | A route is locally valid but violates end-user aggregate intent. | overspend, stranded value, or unwanted market exposure | user-selected bad route | MidnightBundles, Midnight.multicall | assert balance deltas and atomic rollback |
| Flash loan / temporary liquidity | Temporary balances satisfy a check or settle twice. | unbacked liability | fully repaid temporary loan | flashLoan, callbacks, shared token custody | repeat assets and overlap held-token contexts |
| Token integration / nonstandard ERC20 | Token behavior violates integration assumptions. | custody mismatch | weird-token-only behavior | SafeTransferLib and all transfers | use standard tokens unless a broader invariant fails |
| Fee / reward / claim accounting | A claim changes economic context before all positions settle. | claimant extraction or lender loss | known fee race or dust | claimContinuousFee, claimSettlementFee, _updatePosition | compare lazy-touch orderings |
| Market lifecycle / maturity / epoch boundary | The same position receives inconsistent rules at a lifecycle edge. | bad settlement or frozen funds | known maturity-edge class | maturity checks, take, repay, liquidate | combine lifecycle boundary with a distinct mechanism |
| Cross-market / cross-pool contamination | Per-market accounting fails when custody or authorization spans markets. | aggregate deficit | shared-token custody remains backed | market IDs, shared tokens, periphery | assert aggregate token backing across markets |
| Denial-of-service that turns into value loss | A liveness fault blocks a time-sensitive protective action. | forced liquidation or frozen value | self-DoS or chosen-market liveness | ratifier, gate, oracle, callback, route loops | quantify third-party economic loss |
| State desynchronization | Two fields representing one economic state update at different times. | incorrect settlement | view-only difference without impact | position, marketState, bitmap, consumed | touch subsets and compare control execution |
| Invariant/spec mismatch | A proof models fewer paths or identities than runtime code. | false assurance and possibly runtime loss | formal-only gap | certora specs and external summaries | require an executable Solidity path for H/M |
| Business-logic contradiction | Code enforces a weaker promise than protocol intent. | user loss despite valid local calls | market risk disclosed by design | bundles, offers, fee and liquidation paths | state explicit user bound and violate it |
| Governance/admin/configuration boundary | A privileged parameter change invalidates user assumptions. | loss or freeze | operator compromise | role setters, fee setter, market creator | require permissionless escalation or nonconsenting user harm |
| Griefing that escalates to economic harm | A cheap adversarial action forces another actor into a loss-making path. | measurable forced loss | gas-only nuisance | ratifiers, roots, route builders, oracle/gate | quantify attacker cost and victim loss |
| Formal verification blindspots | Summaries, preserved blocks, filters, or vacuity exclude a runtime composition. | false assurance | model-only observation | certora/** | pair spec review with executable regression design |
